Hitting statically linked, small, and good TLS support simultaneously
seems a little tricky.  Which is a shame.  Go produces static
executables and has a pretty nice TLS library (with ED25519 support in
the latest releases), but the executables sure ain't small.  Something
written in C using one of the TLS libraries targetting embedded systems
seems the safest bet.
