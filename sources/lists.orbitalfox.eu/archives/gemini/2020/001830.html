On IRC today there was some discussion about how there should be a way
to tell clients when content is being streamed. This comes on the heels
of Mozz's post and demo of streaming, where he has an IRC-style chat on
Gemini.

gemini://mozz.us/journal/2020-06-21.gmi
gemini://chat.mozz.us

Many of the current clients, such as Kristall, implement a timeout on the
server sending data, it's even part of the Egsam client test. It'd be nice
to be able to know when data is being streamed or when it comes to end, for
things like downloads as well.

It was krixano on IRC who brought this all up and suggested a new 2x status
code for this purpose. Maybe status 22, since 21 used to be for certs?

What do you think?
