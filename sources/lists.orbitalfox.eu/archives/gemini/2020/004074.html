Le mardi 15 décembre 2020, 16:05:14 CET Björn Wärmedal a écrit :
> As for the path that follows, that must be percent encoded in the
> gemtext document. There is no way for a client to know if a path is
> already percent encoded or not, and percent encoding twice breaks the
> link. Consider this:
> => gemini://example.com/why-space-is-%20-in-urls.gmi We see that this
> needs to be percent encoded, but a tool can't reliably tell if it does
> or if it is already.

This is not true, even when using IRI, reserved characters such as spaces HAVE to be percent-encoded.

So, if you see a "%", it is percent encoding. If you want to link to a path containing a percent, you have to percent encode the percent, resulting in %25.

As a result, percent encoding twice does not break the link, as you only percent encode what is not percent encoded already.
