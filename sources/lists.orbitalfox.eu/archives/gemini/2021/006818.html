Hello all,

Im considering gemini interoperability with regards to the felinks browser (given is should hopefully be added to it this year):
=> https://github.com/rkd77/elinks/issues/121 (https://github.com/rkd77/elinks/issues/121)

Im hoping to prioritise using its (rough around the edges) bookmarking facility to access XBEL - the XML Exchange Language.
=> http://pyxml.sourceforge.net/topics/xbel/docs/html/xbel.html (http://pyxml.sourceforge.net/topics/xbel/docs/html/xbel.html)

The justification for this is that I hope to connect it with other information assets :
* Probably using Guile (which is tight with XML, including with regards to SAX parsing)
* Hopefully using an felinks browser within EXWM environment, with Hyperbole - to integrate it better with my workflow

The homepage is already full of examples regarding how it is currently used.
=> http://pyxml.sourceforge.net/topics/xbel/ (http://pyxml.sourceforge.net/topics/xbel/)

However, are there any people in the Gemini community currently:
* Absorbing a lot of XML data in order to generate .gmi content?
* Grapping .gmi content and extracting it into XML.

I would assert that migrating data from HTML centric bookmarking to Gemini centric bookmarking would be a win.

Are there any case studies or suggested workflows from the Gemini community?
